# Security Policy

## Supported Versions

This project currently maintains security fixes for the active production codebase only.

| Version | Supported |
| ------- | --------- |
| Current production deployment | :white_check_mark: |
| Latest repository `main` branch | :white_check_mark: |
| Older snapshots / previous deployments | :x: |
| Unpublished local copies / forks | :x: |

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** report it through public issues, comments, or discussion threads.

Please report it privately through:

- Email: `bicolindigenouspeopleshub@gmail.com`

When reporting, include as much detail as possible:

- A short summary of the issue
- The affected page, file, or feature
- Steps to reproduce
- Any screenshots or proof-of-concept details
- The possible impact
- Your contact information for follow-up

### What to expect

- Initial acknowledgement: within **3 to 7 days**
- Status updates: provided when the report is confirmed, under review, fixed, or declined
- If accepted: the issue will be investigated, prioritized, and patched in the supported version
- If declined: you may receive a brief explanation if the report is out of scope, unsupported, or not reproducible

### Scope

This policy applies to security issues involving:

- Authentication and authorization
- Role-based access control
- Firestore security rules
- User data exposure
- Emergency alert and location-sharing records
- Admin and super admin access
- File or image upload handling
- Any vulnerability that may affect confidentiality, integrity, or availability of the platform

Thank you for helping keep the Bicol Indigenous Peoples Hub safe and trustworthy.
